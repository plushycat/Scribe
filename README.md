# Scribe

Scribe is a lightweight Windows-first SQL\*Plus session recorder. It launches SQL\*Plus as a child process, records a live raw transcript, supports `/scribe` internal commands, and stores session data on disk ‚Äî using only the Python standard library.

## Quick Start

```powershell
scribe
```

Pass SQL\*Plus arguments directly:

```powershell
scribe hr/tiger
```

Use a custom SQL\*Plus executable:

```powershell
scribe --sqlplus C:\oracle\product\11.2.0\dbhome_1\BIN\sqlplus.exe -- hr/tiger
```

If SQL\*Plus requires elevation, relaunch Scribe through UAC:

```powershell
scribe --elevate -- hr/tiger
```

You can also run via Python module:

```powershell
python -m scribe --elevate -- hr/tiger
```

Password prompts are detected automatically and masked with `‚àé`.

## Installation

```powershell
pip install -e .
```

This makes `scribe` available as a global command in any PowerShell window.

To build a standalone executable (no Python required on target):

```powershell
pip install pyinstaller
pyinstaller --onefile --name scribe -m scribe
```

## Startup Options

| Flag | Description |
|---|---|
| `--config PATH` | Path to config file (default: `scribe.config.json`) |
| `--sqlplus PATH` | SQL\*Plus executable path (overrides config) |
| `--elevate` | Relaunch Scribe in an elevated Windows Terminal window |
| `--` | Delimiter for arguments forwarded to SQL\*Plus |

## Internal Commands

Internal commands are consumed by Scribe and are not forwarded to SQL\*Plus.

| Command | Description |
|---|---|
| `scribe status` | Show session info (id, paths, counts) |
| `scribe note <text>` | Attach a note to the current session |
| `scribe export` | Export commands to Markdown (incremental) |
| `scribe search <term>` | Search past session transcripts |
| `scribe history` | Show recent commands |
| `scribe bootstrap` | Re-run SQL\*Plus setup commands |
| `scribe help` | Show this help |

The `/scribe` prefix also works for backwards compatibility.

## Input Editing

Scribe provides a full line-editing experience while typing SQL commands:

| Key | Action |
|---|---|
| Left/Right arrows | Move cursor through the input |
| Up/Down arrows | Navigate command history |
| Backspace | Delete character before cursor |
| Delete | Delete character at cursor |
| Enter | Execute the command |
| Ctrl+C | Interrupt (configurable via `ctrl_c_exit`) |

The SQL prompt (e.g., `SQL> `, `hr@ORCL> `) is captured dynamically from SQL\*Plus and preserved with its original color during redraws.

## Color Output

Scribe uses truecolor hex ANSI codes for terminal coloring. Colors auto-detect (enabled when stdout is a TTY) and can be toggled via config.

SQL input is colorized in real-time as you type ‚Äî keywords light up on word boundary, similar to VSCode.

| Element | Color | Hex |
|---|---|---|
| SQL keywords | Blue | `#61AFEF` |
| Strings | Green | `#98C379` |
| Numbers | Orange | `#D19A66` |
| Operators / `;` | Cyan | `#56B6C2` |
| ORA-/SP2- errors | Red | `#E06C75` |
| Scribe messages | Purple | `#C678DD` |
| SQL prompt | Blue | `#61AFEF` |

Color can be disabled in `scribe.config.json`:

```json
{
  "ui": {
    "color": false
  }
}
```

## Aliases

User-configurable shortcuts that expand before being sent to SQL\*Plus. The expansion is printed in purple so you always see what was substituted.

`cls`, `CLEAR SCREEN`, and similar commands are handled by Scribe directly ‚Äî the terminal clears immediately without relying on SQL\*Plus output.

```json
{
  "aliases": {
    "cls": "CLEAR SCREEN",
    "show tables": "SELECT table_name\nFROM user_tables\nORDER BY table_name;"
  }
}
```

## Export

`/scribe export` appends new commands and notes to a Markdown file. Each export only includes content added since the last export ‚Äî no duplicates.

## Configuration

Default configuration is created at `scribe.config.json` on first run:

```json
{
  "sqlplus_path": "sqlplus",
  "sessions_dir": "sessions",
  "bootstrap_commands": [
    "SET LINESIZE 200",
    "SET PAGESIZE 1000",
    "SET SERVEROUTPUT ON",
    "SET SQLPROMPT '(Scribe) SQL> '"
  ],
  "aliases": {
    "cls": "CLEAR SCREEN",
    "clrscr": "CLEAR SCREEN",
    "wipe": "CLEAR SCREEN",
    "show tables": "SELECT table_name\nFROM user_tables\nORDER BY table_name;"
  },
  "ignore_export_commands": ["CLEAR SCREEN", "cls", "clrscr", "wipe"],
  "export": {
    "title": "Session"
  },
  "ctrl_c_exit": true
}
```

| Key | Default | Description |
|---|---|---|
| `ctrl_c_exit` | `true` | When `false`, Ctrl+C is ignored during input. Use `exit` or `quit` to leave. |

## Session Files

Each session produces three files under `sessions/`:

| File | Contents |
|---|---|
| `YYYY-MM-DD_HHMMSS.log` | Raw canonical transcript (never discarded) |
| `YYYY-MM-DD_HHMMSS.json` | Session metadata (commands, notes, exports) |
| `YYYY-MM-DD_HHMMSS.md` | Markdown export (generated by `/scribe export`) |

## Requirements

- Python 3.10+
- Windows (primary; elevation and key-handling are Windows-specific)
- Oracle SQL\*Plus or any compatible command-line client
- No external Python dependencies (stdlib only)
